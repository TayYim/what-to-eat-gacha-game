# 02 Requirements and Product Design

## Goal

Turn the researched product direction into a scoped PRD, a minimal MVP, and an
implementation-ready design.

## PRD Requirements

Create a PRD covering:

1. Product goal
2. Target users
3. Core user stories
4. Functional requirements
5. Non-functional requirements
6. Key workflows
7. Data model or core entities, if applicable
8. System boundaries
9. Explicit non-goals
10. Success criteria
11. Risks and assumptions

Clearly separate features into:

- Must-have
- Should-have
- Nice-to-have

Use the PRD template when helpful:
`templates/prd.md`.

## MVP Scope

Define the smallest prototype that can demonstrate the core product value. Do
not overbuild. Exclude anything that does not materially improve the prototype's
ability to prove the main workflow.

## Product Design

Brainstorm and document:

1. Overall user experience
2. Main screens or interfaces, if applicable
3. Backend or system architecture
4. Data flow
5. Major modules
6. External dependencies
7. Testing strategy
8. Security, privacy, and reliability considerations where relevant

## Milestones

If the project is large, split the design into milestones. Each milestone must
have:

- Clear goal
- Deliverable
- Acceptance criteria
- Regression-risk checklist

## Review Gate

Before implementation, perform a critical review of the PRD and design. Check
for overbuilt scope, missing user workflows, weak acceptance criteria, unclear
data boundaries, unverified claims, and unrealistic implementation assumptions.
