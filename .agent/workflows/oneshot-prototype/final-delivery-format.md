# Final Delivery Format

At the end of the project, provide:

1. Final product summary
2. Core features implemented
3. Architecture summary
4. Setup and usage instructions
5. Test results
6. Demo instructions
7. Known limitations
8. Remaining TODOs
9. Review scores and reviewer summaries
10. Explanation of whether the prototype meets the original idea
11. Recommended next milestone, if further development is needed
12. Cleanup status for test-only services, temporary artifacts, documentation,
    and Git commit boundaries

Use the final delivery template when helpful:
`templates/final-delivery.md`.
