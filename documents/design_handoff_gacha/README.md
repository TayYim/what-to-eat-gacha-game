# Handoff: 今天吃啥 · 召唤 (What-to-eat Gacha)

A mobile-first "decide what to eat" app reframed as a **gacha / destiny-summon** experience.
Two ways to decide: a **card-pull gacha** (single / ten-pull with rarity, reveal animation,
pity floor) and a **fortune wheel** (category wheel + food wheel).

> **THIS IS A DESIGN REFERENCE, NOT PRODUCTION CODE.**

> and behavior. Your job is to **recreate this design in the project's real codebase**

**Fidelity: HIGH (hi-fi).** Colors, typography, spacing, motion timings, and the procedural
artwork are all final. Match them precisely. Every measurement, hex value, and easing curve in
this document is authoritative.

## The artwork is procedural — port the generators, do not redraw

There are **no image assets.** Every food/category icon and the background are generated by code.
To "replicate exactly," you must port these two generators **verbatim** (same constants, same
math) so output is pixel-identical:

### 1. Constellation glyphs — `glyph(seed, color, size)`
Each dish and category gets a unique little **constellation** SVG: 4–6 nodes placed by a seeded
PRNG, joined by a faint stroke, each node rendered as a soft colored halo + white core + colored
center dot (the first node is larger). Deterministic from the seed string, so the same dish always
produces the same constellation.

- Hash: **FNV-1a 32-bit** — `hashStr(s)` (offset 2166136261, prime 16777619).
- PRNG: a **mulberry32-style** generator seeded by that hash — `prng(seed)`.
- Node count `n = 4 + floor(rnd()*3)`; each coord in `[16, 84]` on a `0 0 100 100` viewBox.
- Path stroke `color`, width 1.1, opacity 0.5, round caps/joins.
- Per node: halo `circle r=rad*2.7 fill=color opacity .14`, white core `circle r=rad`,
  colored dot `circle r=rad*0.5 fill=color`. First node `rad=3.6`, others `1.8 + rnd()*1.7`.
- **Seeds used:** `"cat-<categoryId>"`, `"food-<foodId>"`, `"banner-destiny"`.

Copy `hashStr`, `prng`, and `glyph` exactly from `source/Gacha.dc.html`. Cache by
`seed|color|size` as the prototype does.

### 2. Starfield background — `canvasRef` loop
Full-screen `<canvas>` behind everything (`z-index:-1`, `pointer-events:none`).
- Star count `= max(40, round(W*H/8500 * particleDensity))`.
- Each star: radius 0.4–1.9, color from `["#ffffff","#ffe9b8","#cdd6ff","#e7c8ff"]`, twinkles
  via sine on opacity (0.35–1.0), drifts slowly downward (wraps at bottom).
- Plus ~16% as many gold "dust" motes (`#ffcf6b`) rising upward, opacity 0.5.
- `devicePixelRatio` capped at 2; honor `prefers-reduced-motion` (CSS already freezes animations).

Behind the canvas sit two fixed gradient layers (see Design Tokens → Background).

## Screens / Views

The whole app is a single mobile column, **max-width 480px, centered**, min-height `100dvh`,
padding `14px 14px 116px` (the bottom pad clears the fixed action bar). There are two primary
tabs plus overlays.

### A. Header (always visible)
- Left: a 42×42 rounded-13px tile (gradient gold→violet, inset gold ring, `tw-float` 5s bob)
  containing the glyph **食** in Noto Serif SC 900/20px, colored `accent`, with an accent glow.
- Title block: `今天吃啥 ·(accent) 召唤` — Noto Serif SC 700/19px, letter-spacing .04em.
  Sub-label `DESTINY GACHA` — Cinzel 600/9.5px, letter-spacing .34em, color rgba(200,206,255,.55).
- Right: 40×40 rounded-12px ghost button, glyph **史** (history), opens the history drawer.

### B. Tab switcher
Two-column grid, gap 8px, 5px padding, rounded-16px, bg rgba(10,8,28,.55), inset hairline.
Each tab is a stacked label: Chinese (Noto Serif SC 700/15px) over Latin (Cinzel 600/8px,
letter-spacing .2em, opacity .7), min-height 54px, rounded-13px.
- **Active:** text #1a0f33 on gradient `135deg, accent → #ff9ecf 70%`, soft pink shadow + inset
  white hairline.
- **Inactive:** text rgba(220,224,255,.7), transparent.
- Tabs: `卡池抽卡 / SUMMON` (gacha) and `命运转盘 / WHEEL` (wheel). Transition all .25s.

### C. Category chips (filter, both tabs)
Horizontal scroll row, gap 8px, padding `13px 1px 3px`. One pill per category (18 total).
Pill: inline-flex, gap 6px, min-height 34px, padding `0 12px`, radius 99px, transition .2s.
Left side shows the category's constellation glyph (16px) in the category color.
- **On:** text #fff, bg `color-mix(in srgb, <catColor> 22%, transparent)`, inset 1px ring in catColor.
- **Off:** text rgba(200,206,255,.45), bg rgba(255,255,255,.03), faint inset ring.
Toggling a chip removes/adds that category from the active pool (affects both gacha pool and wheel).

### D. Gacha tab (`卡池抽卡`)
1. **Banner card** — rounded-20px, padding 18px, gradient `155deg rgba(48,32,96,.78)→rgba(20,14,46,.72)`,
   inset gold hairline + deep drop shadow, two soft radial glows, a large floating `banner-destiny`
   glyph top-right (150px, `tw-float` 7s).
   Copy: `LIMITED BANNER` (Cinzel 600/9.5px, .32em, accent) · `限定 · 今日卡池` (Noto Serif SC 900/26px)
   · body `十连必出 4★ 稀有 及以上 · 抽取权重由稀有度决定` (12.5px, rgba(214,218,255,.7)).
2. **Rarity weights card** — rounded-16px, header `RARITY WEIGHTS · 稀有度权重` + `十连保底 4★+`.
   Three equal columns showing the weights: **R · 优良 = 94**, **SR · 稀有 = 5**, **SSR · 传说 = 1**.
   Each cell: tier-tinted bg (`color-mix … 12–14%`), inset tier-colored hairline, tier label in
   Cinzel 800/11px, the number in Noto Serif SC 900/19px white + `权重` caption.
3. **Last summon card** (only after a pull) — horizontal: 56×72 glyph tile + name + stars,
   tinted by the best result's tier color. Header `上次召唤 · <TIER>`.
4. **Fixed action bar** (bottom, z-40, gradient fade to bg) — two buttons in a `1fr 1.45fr` grid:
   - `单抽 / ×1` — ghost (rgba(255,255,255,.07), inset hairline), min-height 58px, radius 17px.
   - `十连召唤 / SUMMON ×10` — gradient `135deg accent → #ff9ecf 55% → #c08cff`, text #1a0f33,
     Noto Serif SC 900/17px, big pink shadow.
   - Both go to 40% opacity + non-interactive when the pool is empty (all chips off).

### E. Wheel tab (`命运转盘`)
1. **Sub-switch** — two buttons `大类转盘` (category wheel) / `食物转盘` (food wheel),
   active = gradient `135deg accent → #ff9ecf`, text #1a0f33.
2. **The wheel** — a 320px (max 100%) square, centered.
   - Outer blurred conic halo, inset gold ring + inner shadow, a dashed ring rotating slowly
     (`tw-ring` 26s).
   - **Pointer:** a triangle at top (`border-top: 30px solid accent`, drop-shadow).
   - **Disc:** `width 84%`, conic-gradient built from the slice colors (`from -90deg`), inset white
     hairline. Rotation is animated via inline `transform: rotate(<deg>)`; during a spin the
     transition is `1750ms cubic-bezier(.12,.74,.06,1)`, otherwise `.4s ease`.
   - **Markers:** one per entry, positioned on a radius, each a rounded white tile holding the
     entry's glyph. Marker size is **adaptive** to entry count:
     `mSize = max(13, min(42, floor(660 / n)))`, icon `= round(mSize*0.56)`,
     radius `R = round(134 - mSize/2 - 5)`. Each marker is rotated to its slice angle and the
     icon counter-rotated so it stays upright. Selected entry (after spin) gets a white ring + glow.
   - **Hub:** a 23% center disc, radial dark fill, gold ring, glyph **命** (Noto Serif SC 900/17px, accent).
3. **Result card** — rounded-18px, header `RESULT`. Empty state: `别纠结了 —— 命运已就位，转动它。`
   Filled: glyph tile + name (Noto Serif SC 900/22px) + meta line + stars. For food results the meta
   is `<category> · <tier label> · 权重 <w>`; for category results it's `今日大方向`.
4. **Spin button** — full width, min-height 56px, same accent→pink→violet gradient.
   Label: `转动命运转盘` / `抽取当前食物池` / `命运转动中…` while spinning. Below: `已从 N 个候选中抉择`.

### F. Summon overlay (gacha pull)
Full-screen fixed (z-80), radial dark backdrop. Two phases:
- **Summoning** (`phase==='summoning'`, ~1700ms): centered floating banner glyph, a pulsing ring
  (`tw-pulse` 1.4s) + counter-rotating dashed ring (`tw-ringR` 3s) + a rising light beam
  (`tw-beam` 1.5s). Aura color is driven by the **highest rarity in the pull** (gold/violet/blue).
  Copy `SUMMONING` (Cinzel, .4em, glowing) over `命运正在汇聚…`.
- **Revealing** (`phase==='revealing'`): header `十连召唤结果 · k/10` (or `召唤结果` for single) and a
  `跳过 ⏩` skip button. Cards flip in one by one.
  - **Grid:** single pull → one large centered card (`min(64vw,220px)`); ten-pull → `repeat(5,1fr)`
    grid, gap 8px (two rows of 5). Card aspect-ratio .72.
  - **Face-down cover:** gradient `155deg #2a1d63→#140e2e`, inset gold hairline + inner glow, glyph
    **食** centered (gold).
  - **Face-up front:** `tw-pop` flip-in (.5s), bg `160deg color-mix(<tier> 24%, #160f36) → #100a28`,
    inset 1.5px tier ring (plus tier glow for 4★+). Top-left tier code, a one-shot diagonal shine
    sweep (`tw-shine`), the dish glyph (40px / 72px single), name (Noto Serif SC 700/13px), category
    caption, and stars.
  - **Reveal cadence:** SSR card dwell 760ms, SR 560ms, R 300ms. Each SSR triggers a gold screen
    flash (`tw-flash` 700ms, z-90). `跳过` reveals all remaining instantly.
  - When all revealed: footer `完成` (close) + `再抽十连` (pull ten again).

### G. History drawer
Bottom sheet (z-71, max-width 480, max-height 74vh), slides up (`tw-drawer` .32s
`cubic-bezier(.2,.8,.2,1)`) over a blurred scrim (z-70, tap to close). Grab handle, title
`命运记录`, `最近 N 次`. Each row: a tier/category-colored capsule bar + label + meta + tier/WHEEL
code. Empty state `尚无记录 —— 去召唤你的第一餐吧。` Holds the most recent **30** events
(both pulls and spins).

## Interactions, behavior & timing

| Action | Result |
|---|---|
| Tap tab | switch `mode` between `gacha` / `wheel` |
| Tap chip | toggle category in `active`; recomputes pool, wheel entries, disables actions if pool empty |
| 单抽 / 十连召唤 | `doDraw(1|10)` → overlay: summoning 1700ms → reveal one-by-one |
| 跳过 | reveal all remaining cards instantly; still flashes if a SSR is present |
| 完成 | close overlay, reset draw state |
| 再抽十连 | immediately pull another ten |
| 大类/食物转盘 | set `wheelMode`, clear current result |
| Spin | rotate disc 5 full turns + land selected slice at top over 1750ms, then show result |
| 史 button / scrim | toggle history drawer |

**Gacha draw logic** (`pickWeighted` over the active pool):
- Per draw, weighted random by `rarityWeight(r)` → R=94, SR=5, SSR=1.
- **Ten-pull pity floor** (gated by the `tenPullGuarantee` prop, default on): if a ten-pull contains
  no 4★+ result, the 10th slot is replaced with a weighted pick from the 4★+ subset.
- `auraRarity` = max rarity drawn (drives the summoning aura color).

**Wheel logic:**
- Category wheel: uniform random among active categories.
- Food wheel: `pickWeighted` over the active pool (same rarity weighting), then the disc lands on
  that entry. Markers shrink as entry count grows (formula above) so 100+ dishes still fit.

**History:** every pull (logs best card; label `<name> 等 N 项` for multi) and every spin is unshifted
onto a list capped at 30.

**Motion:** all transitions/animations collapse to ~0ms under `prefers-reduced-motion: reduce`
(already handled in the prototype's `<style>`).

## State model

```
mode:        'gacha' | 'wheel'
wheelMode:   'category' | 'food'
phase:       'idle' | 'summoning' | 'revealing'     // gacha overlay
active:      Record<categoryId, boolean>            // chip filters (all true initially)
rotation:    number (deg, accumulates)              // wheel
spinning:    boolean
wheelResult: { id, name, color, meta, isCat, r } | null
draws:       { item: Food }[]                       // current pull
revealedCount: number                               // cards flipped so far
auraRarity:  number
flash:       color string | null                    // SSR screen flash
lastDraws:   Food draws of the last completed pull  // powers "last summon" card
history:     { id, label, meta, code, color }[]     // capped 30
showHistory: boolean
```

Timers must be cleared on unmount / overlay close / skip (the prototype tracks them in `_timers`).
